In Node.js simple things become complicated, because of its asynchronous nature. Most people realized this as soon as they start iterating over stuff. In Node most things are asynchronous, and most IO operations take a callback that is executed as soon as the data is available. And the actual execution returns immediately. This example hopefully makes it clear.

 

This sort of behaviour surprises a lot of people, who come from synchronous programming background. If you trpe console.log(data) a few seconds later, it will actually show the correct values on the node commandline. Leading to strange results.

 

There are multiple ways of dealing with this situation.

  1. Using libraries build on top of node-fibres like Node-Sync
  2. Use Counters/ nextTick to schedule s
  3. Rolling your own abstraction
  4. Use streamline.js that generates javascript to deal with this issue.
